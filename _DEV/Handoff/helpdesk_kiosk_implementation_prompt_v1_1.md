# 헬프데스크 키오스크 모드 구현 프롬프트 (v1)

> Claude Code 전용 구현 지시문. 이 턴에서는 **구현만** 한다. 검증·E2E·체크포인트는 별도 턴 별도 프롬프트에서 다룬다.

---

## §0 역할

너는 `reverve9/moosan` 레포의 React + Vite + TypeScript + Supabase 코드베이스에서 작업한다. 현재 브랜치는 `dev`. 이번 작업은 헬프데스크 부스용 **키오스크 모드** 신설이다.

---

## §1 작업 범위

### 1-1. 배경 (한 줄 요약만 읽고 넘어가도 됨)

기존엔 헬프데스크에서 손님이 메뉴를 구두로 말하면 직원이 어드민에 입력 → 카드/현금 결제 → 주문 인서트 순서였다. 이걸 **손님이 직접 큰 터치모니터(스탠바이미 Go 27인치, 어드민 노트북에 확장 연결)에서 메뉴를 골라 전번 입력 → "결제 요청" → 어드민 대기 큐 → 직원이 별도 카드 단말기/현금 받고 어드민에서 "결제완료" 처리** 흐름으로 바꾼다.

핵심: 헬프데스크 흐름은 **토스 PG 미사용**. 카드는 별도 오프라인 단말기, 현금은 직접 수령. 어드민에서 결제수단·금액 기록만 한다.

### 1-2. 라우트 및 화면 구조

신규 라우트 `/kiosk` 를 만든다. 단일 페이지 안에서 step state 로 단계 전환 (별도 라우트 분기 X).

| step | 화면 | 비고 |
|---|---|---|
| `menu` | 3컬럼 가로 레이아웃 (좌: 부스 리스트, 중: 메뉴 그리드, 우: 장바구니) | 풀스크린 1920x1080 기준 |
| `phone` | 전번 입력 (큰 숫자 키패드) | 모달 또는 풀스크린 오버레이 |
| `waiting` | "직원에게 카드/현금을 제시해주세요 - 대기번호 #N" | Supabase realtime 으로 결제완료 감지 |
| `done` | "결제 완료 - 5초 후 처음으로 돌아갑니다" 카운트다운 | 자동 리셋 |

기존 모바일 PWA 의 메뉴 카드·장바구니 로직·부스/메뉴 데이터 페칭 훅 등 **재사용 가능한 것은 재사용**. 가로 레이아웃 래퍼만 신규 작성.

### 1-3. 디자인 톤 (이 챗 세션의 일관된 디자인 원칙)

- 타이포·아이콘 디테일 신경 쓸 것. Lucide 아이콘 strokeWidth `1.2` 기본
- 터치 친화적: 버튼 최소 높이 60px, 폰트 충분히 크게 (노년층 사용 고려)
- 색 대비 명확, 결제 대기 화면은 시인성 최우선
- 헤더 우측 상단에 **"처음으로"** 버튼 상시 노출 (단, `waiting` step 에서는 숨김)

### 1-4. DB 스키마 변경

`orders` 테이블에 컬럼 2개 추가:

- `payment_channel`: `'app' | 'helpdesk'` — NOT NULL, default `'app'`
- `payment_method`: `'card' | 'cash' | null` — 헬프데스크만 채움, 앱 결제는 null 유지

`status` 열에 신규 값 `'payment_pending'` 허용. 흐름:
- `payment_pending` (헬프데스크 키오스크에서 결제 요청 후, 직원 처리 전)
- → `paid` (직원이 어드민에서 결제완료 처리)
- 이후 기존 흐름 그대로 (`preparing` → `ready` → `picked_up` 등)

기존 앱 결제 흐름의 status 전이는 **건드리지 말 것**.

### 1-5. 결제 요청 플로우

키오스크 `phone` step 에서 "결제 요청" 클릭 시:
- `orders` insert: `status='payment_pending'`, `payment_channel='helpdesk'`, `payment_method=null`
- 인서트 후 `waiting` step 으로 전환, 화면에 주문번호(또는 짧은 대기번호) 표시

PG 결제창 호출 **금지**.

### 1-6. 어드민 결제 대기 큐

어드민 대시보드에 **"결제 대기"** 섹션 신설 (또는 기존 결제 도우미 영역이 이미 있다면 그 자리에 통합):

- 표시 대상: `status='payment_pending' AND payment_channel='helpdesk'` 인 주문
- Realtime 구독으로 신규 주문 즉시 표시
- 각 주문 클릭 시 결제 처리 모달:
  - 결제수단 선택: `[카드] [현금]`
  - "결제완료" 버튼 클릭 → `status='paid'`, `payment_method` 채움, `paid_at=now()` 업데이트
- 결제완료 후 매장 대시보드로 정상 흘러가야 함 (기존 흐름과 동일)

### 1-7. 리셋 로직 3종

**(a) 무동작 타임아웃**
- 키오스크 `menu` 또는 `phone` step 에서 3분 동안 터치 없으면 자동 리셋
- 리셋 5초 전 "자동으로 처음 화면으로 돌아갑니다" 토스트 표시
- `waiting` step 에서는 **타임아웃 적용 안 함** (직원 처리 대기 중이므로)

**(b) 손님 "처음으로" 버튼**
- `menu` / `phone` step 에서 헤더 우측 상단에 상시 노출
- 클릭 시:
  - 장바구니 비어있으면 즉시 메인으로
  - 장바구니에 항목 있으면 확인 모달: "담은 메뉴를 모두 비우고 처음부터 다시 시작할까요?" / `[취소]` `[다시 시작]`
- `waiting` step 에서는 버튼 자체를 숨김 (이미 DB 인서트 후라 직원 처리만 가능)

**(c) 직원 강제 리셋**
- 어드민에서 "키오스크 초기화" 버튼 클릭 → 키오스크 즉시 메인 리셋
- Supabase Realtime **Broadcast 채널** 사용: 채널명 `kiosk:helpdesk-1`, 이벤트 `force-reset`
- DB 거치지 않고 broadcast 만으로 처리 (단순화)
- 키오스크 측: 채널 구독 후 `force-reset` 수신 시 step `menu` 로 리셋 + 장바구니 클리어
- 어드민 측: 확인 모달 없이 즉시 broadcast, 어드민 화면에 토스트만 "키오스크 초기화 요청 보냄"

### 1-8. 결제완료 → 키오스크 자동 전환

직원이 어드민에서 `payment_pending` 주문을 `paid` 처리하면:
- 키오스크는 해당 주문의 `status` 변경을 Realtime 으로 감지
- `waiting` → `done` step 으로 자동 전환
- 5초 카운트다운 후 `menu` 로 자동 리셋 (장바구니 비움)

### 1-9. 통계 어드민 채널별 분리

기존 통계 페이지에 `payment_channel` 기준 매출 분리 표시 추가:
- 앱 결제 매출 / 헬프데스크 매출 / 합계
- 헬프데스크 내에서 카드 vs 현금 비율도 별도 분리
- 매장별 매출엔 두 채널 합산 (정산 기준)

기존 시각화 컴포넌트 그대로 쓰되 데이터 그룹핑만 추가.

### 1-10. 풀스크린·키오스크 모드 가이드 (운영용)

README 또는 `docs/kiosk-operation.md` 에 짧게 추가:

- Chrome 키오스크 모드 실행 예시:
  ```
  chrome --kiosk --app=https://dev.musanfesta.com/kiosk
  ```
- 운영 모드(prod) URL 및 dev URL 양쪽 안내
- 노트북에서 두 번째 브라우저 창을 스탠바이미 모니터로 드래그 후 풀스크린 띄우는 절차

## §2 금지 사항

- ❌ **토스 PG 결제창 호출** — 헬프데스크 흐름은 PG 미사용
- ❌ 기존 모바일 PWA 결제 흐름 코드 수정 (`payment_channel='app'` 흐름은 전혀 건드리지 않음)
- ❌ 매장 대시보드 흐름 변경 (`status='paid'` 이후는 기존과 100% 동일하게)
- ❌ 어드민 통계의 기존 컴포넌트 구조 변경 (그룹핑만 추가, 시각화 자체는 재사용)
- ❌ 새 라우트를 `/kiosk` 외의 경로로 만들지 말 것
- ❌ 키오스크 페이지에 `<link rel="manifest">` PWA 설치 유도 메타 등 영향 없는 것 추가 금지
- ❌ 손님이 `waiting` step 에서 빠져나오는 경로 만들지 말 것 (직원 강제 리셋·결제완료 자동전환·새로고침만 허용)

## §3 착수 전 확정 사항 (이 챗에서 결정 완료, 추가 질문 금지)

| 항목 | 결정값 |
|---|---|
| 헬프데스크 운영 대수 | **1대** (스탠바이미 Go 27인치, 어드민 노트북에 확장 연결) |
| 키오스크 라우트 | `/kiosk` (단일 페이지 + step state) |
| 화면 기준 해상도 | 가로 1920x1080 풀스크린 |
| PG 사용 여부 | **미사용** (헬프데스크 결제는 토스 PG 거치지 않음) |
| 결제 수단 | `card` (별도 단말기) / `cash` (현금 수령) |
| 무동작 타임아웃 | 3분 (단, `waiting` step 제외) |
| 결제완료 후 자동 리셋 | 5초 카운트다운 후 메인 리셋 |
| 직원 강제 리셋 채널 | Supabase Realtime Broadcast (DB 거치지 않음) |
| 채널명 | `kiosk:helpdesk-1` / 이벤트 `force-reset` |
| 손님 알림 | 기존 PWA 흐름 그대로 (전번 입력 → 알림톡/푸시) |
| 통계 채널 분리 | 앱 / 헬프데스크(카드·현금) / 합계 |
| 작업 브랜치 | `dev` |

판단 모호한 케이스가 나오면 **착수 보고**에 명시하고 사용자(이 챗) 결정 대기. 자체 추정으로 진행 금지.

## §4 커밋 방식

- 작업 단위로 분할 커밋 (예: `feat: /kiosk 라우트 신설 및 3컬럼 레이아웃`, `feat: orders 테이블 payment_channel/payment_method 컬럼 추가`, `feat: 어드민 결제 대기 큐`, `feat: 키오스크 리셋 3종`, `feat: 통계 채널별 분리` 등)
- 커밋 메시지 한글 가능, conventional commit prefix(`feat:`, `fix:`, `chore:`, `refactor:`) 권장
- `dev` 브랜치 직접 커밋 (musanfesta-dev Vercel 자동 배포)
- 마이그레이션 SQL 은 별도 파일로 (`supabase/migrations/` 또는 레포 컨벤션 따름)

## §5 핸드오프 예상 출력 (구현 완료 시 보고 형식)

구현 완료 후 다음 항목 정리해서 보고:

1. **변경/신규 파일 목록** (경로 + 한 줄 설명)
2. **마이그레이션 SQL** (별도 코드블록으로 정리 — 사용자가 Supabase Dashboard 에서 직접 실행 또는 CLI 실행)
3. **접속 URL**: dev 환경 키오스크 URL, 어드민 결제 대기 화면 진입 경로
4. **운영 매뉴얼**: Chrome 키오스크 모드 실행 명령어, 두 번째 모니터로 띄우는 절차
5. **자체 점검에서 발견된 한계·이슈** (예: realtime 지연, 특정 브라우저 호환 등) — 사용자 검증 전 미리 공유
6. **불확실해서 사용자 결정 필요했던 항목** (있다면)

## §6 다음 블록 예고

이 구현이 완료되면 **별도 검증 프롬프트**가 전달된다. 검증 프롬프트에서는:
- 사용자가 직접 실행할 DB 쿼리·UI 점검 절차·E2E 시나리오가 명시됨
- Claude Code 는 그 턴에서 코드 수정 금지, 보고만
- 검증 결과는 사용자가 이 챗에 전달 → 다음 스텝 결정

따라서 이번 턴에서는 자체 E2E 까지 시도하지 말고 **구현 완료 보고에서 멈출 것**.
