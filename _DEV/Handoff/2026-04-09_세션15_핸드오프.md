### I. 버그 수정 — elapsed 측정 기준을 paid_at 으로

**문제**: 부스 대시보드 / 어드민 모니터의 1분 alert 가 `orders.created_at` 기준이라, 사용자가 "결제하기" 를 누른 시점부터 측정됐다. 결제수단 선택 / 본인 인증 / 토스 승인까지 30~40초가 elapsed 에 포함되어, 부스에 도달했을 때 이미 alert 직전 (남은 ~20초) 이 됨.

**원인**: `orders` 테이블에 `paid_at` 컬럼이 없었음. `payments.paid_at` 만 존재. 부스/어드민 fetch 가 orders 만 select 해서 결제 완료 시각을 알 길이 없었음.

**해결**: `orders.paid_at` 컬럼 신설 + `markPaymentPaid` 가 status='paid' 와 함께 채움. 부스/어드민 elapsed 계산을 paid_at 기준으로 전환.

**마이그 18** (`_DEV/Seeds/18_orders_paid_at.sql`)
- `orders.paid_at TIMESTAMPTZ NULL` 추가
- 기존 paid 이력 행 backfill: `UPDATE orders SET paid_at = payments.paid_at WHERE ...`
- 부분 인덱스 `idx_orders_paid_at` (paid_at IS NOT NULL)

**코드 변경**
- `src/lib/orders.ts` `markPaymentPaid` — orders update 에 `paid_at: now` 추가
- `src/lib/boothMonitor.ts` — `MonitorOrderRow.created_at → paid_at`, `MonitorBoothSummary.oldestCreatedAt → oldestPaidAt`, 정렬 기준 paid_at
- `src/pages/admin/AdminMonitor.tsx` — alertCount / 카드 elapsed / 시간 표시 모두 paid_at 참조
- `src/pages/booth/BoothDashboardPage.tsx` — `BoothOrderCard.orderCreatedAt → orderPaidAt`, elapsed/정렬/시간 표시 모두 paid_at. NULL fallback `?? created_at` (마이그 적용 직후 안전망)
- `src/types/database.ts` — orders 타입에 paid_at 추가

**효과**
| 시점 | 변경 전 | 변경 후 |
|---|---|---|
| 사용자 "결제하기" 클릭 | elapsed 측정 시작 | (측정 안 함) |
| 토스 결제수단/인증 ~30s | elapsed 진행 | (측정 안 함) |
| 토스 승인 → orders.paid_at | (이미 30s 진행) | **elapsed 측정 시작** |
| 부스 도달 | 남은 ~20초 | 남은 60초 (full) |
