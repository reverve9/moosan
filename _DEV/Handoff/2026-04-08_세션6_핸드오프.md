## 5. musan / food 데이터 모델 + 페이지

### 신규 테이블 (`_DEV/Seeds/05_musan_food.sql`)
- `festival_events` — musan 개막식/폐막식/기타 프로그램 (kind: opening|closing|program, 참가신청 X)
- `festival_guests` — musan 스페셜 게스트 (사진 + 이름 + 소개 + 외부링크)
- `food_booths` — food 매장 (썸네일 + 갤러리 + booth_no)
- `food_menus` — booth 별 메뉴 (이름 / 가격 / 설명 / image_url / is_signature)
- `festivals.layout_image_url` 컬럼 추가 (food 부스 평면도)
- 각 테이블: `updated_at` 트리거 / `(festival_id, sort_order)` 인덱스 / RLS public select + 임시 admin all

### food 카테고리 (`_DEV/Seeds/06_food_categories.sql`)
- `food_booths.category` TEXT, CHECK (`korean | chinese | japanese | fusion`)
- 기존 시드 행은 모두 `korean` 으로 채움

### food 더미 25개 시드 (`_DEV/Seeds/07_food_booths_dummy.sql`)
- 한식 7 / 중식 7 / 일식 7 / 퓨전 4 = 25 매장
- 각 매장 2 메뉴 (1 signature + 1 regular) = 50 메뉴
- DELETE → INSERT 패턴 (idempotent)
- ⚠️ 사용자가 어드민에 직접 입력할 수도 있으므로 **재실행 시 기존 booth 모두 날아감** 주의

### Database 타입 등록 (`src/types/database.ts`)
- 4개 신규 테이블 모두 `Database['public']['Tables']` 에 Row/Insert/Update/Relationships 등록
- 별도 standalone 타입 `src/types/festival_extras.ts` 도 유지 (lib 에서 import)
- ⚠️ **로컬 검증은 `tsc --noEmit` 말고 `npm run build` 로** — project references 차이로 noEmit 가 통과해도 vercel 빌드(`tsc -b`) 는 실패할 수 있음. 한 번 그 이유로 vercel 두 번 연속 실패함

### Lib fetch 함수 (`src/lib/festival.ts`)
- `fetchFestivalEvents(festivalId, kind?)` — kind 필터 (단일 또는 배열)
- `fetchFestivalGuests(festivalId)`
- `fetchFoodBooths(festivalId)` → `FoodBoothWithMenus[]` (booths + menus join)

### MusanSections (`src/components/musan/MusanSections.tsx`)
- 3개 섹션 한 컴포넌트:
  1. 개폐막식 일정 카드

### FoodSections (`src/components/food/FoodSections.tsx`)

- ⚠️ **사용자가 카드 컴팩트화를 명시적으로 요구** — "썸네일은 작게 좌측에", "한 눈에 볼 수 있게 카드 높이 최소화", "텍스트 4행 처리 거부". 카드 디자인 추가 변경 시 이 원칙 유지.
- 카드 클릭 → **편집 모달**:
