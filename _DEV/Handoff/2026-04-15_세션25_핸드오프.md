## 8. 설계 결정 메모

### QR URL 에 `boothId` (UUID) 를 그대로 쓴 이유
짧은 `booth_no` (예: `A01`) 로 쓸 수도 있지만:
- UUID 가 이미 unique
- FoodSections 에서 `booths.find(b => b.id === boothId)` 로 O(N) 탐색, N≤수십이라 성능 무관
- 부스 번호 재배치해도 QR URL 이 깨지지 않음 (QR 재인쇄 불필요)

### customer origin 계산을 env 아닌 런타임으로 한 이유
- 로컬 dev (`admin.localhost:5173`) / 프로덕션 (`admin.musanfesta.com`) 둘 다 동일 규칙으로 동작
- 별도 env 관리 포인트 불필요
- 규칙: `admin.` / `booth.` prefix + `-admin.` / `-booth.` infix 제거

### QR canvas 이중 렌더 (180px + 1024px)
- 화면 표시용 작은 QR 은 CPU/메모리 적게
- 다운로드 시 고해상도 필요 → 숨겨진 1024px canvas 를 CSS `offscreen` 클래스로 off-viewport 배치
- `<a download>` + `canvas.toDataURL('image/png')` 로 즉시 다운로드
