## 5. 결제 흐름 테스트 — `vercel dev` 필요

`npm run dev` 로는 `/api/payments/confirm` 가 404. 결제 끝까지 테스트하려면:

```bash
npm i -g vercel       # 처음 한 번
vercel link           # 프로젝트 연결, 처음 한 번
vercel dev            # 5173 대신 3000 포트로 띄워짐
```

→ http://localhost:3000 으로 접속 → 음식 페이지 → 메뉴 담기 → 카트 → 주문하기 → 토스 테스트 카드 → success → `/order/:id` 까지 한 흐름.

### 흔한 에러
- `404 /api/payments/confirm` → `vercel dev` 가 아니라 `npm run dev` 로 띄움
- `500: TOSS_SECRET_KEY is not configured` → `.env` 에 시크릿 키가 없거나 `VITE_` 가 잘못 붙음. `vercel dev` 재시작
- `결제 금액이 일치하지 않습니다` → CheckoutSuccessPage 가 URL 의 amount 와 DB total_amount 비교, 부분 INSERT 후 재시도 등 데이터 꼬임. 카트 비우고 다시 담기

### 사용자 보고
이번 세션 끝에 **장바구니 담기까지 완벽 테스트 성공**. 결제 흐름 (`vercel dev` 로 토스 SDK → `/checkout/success` → `/order/:id`) 은 **미테스트**. 다음 머신에서 위 절차로 한 번 끝까지 굴려볼 것.
