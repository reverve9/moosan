### 4-2. ⭐ 작업 B — 키오스크 menu step 좌측 매장 카테고리 분류

**작업 A 완료 후 또는 사용자 우선순위에 따라 진행.**

#### 현재 상태
`src/pages/kiosk/steps/MenuStep.tsx` 의 좌측 부스 리스트(320px 컬럼)는 단순 평면 리스트로, 모든 활성 부스를 `booth_no` 순으로 나열. 매장 수가 많을 때 손님이 빠르게 찾기 어려움.

#### 변경 방향
좌측 컬럼 상단에 **카테고리 필터 칩** 또는 **카테고리별 그룹 헤더**를 추가. 핸드오프 §1-3 디자인 톤(노년층 + 시인성) 고려하여 그룹 헤더 방식 권장 (필터 칩은 추가 클릭 1번 필요).

**선택지 2가지** (사용자 결정 필요):
- **(a) 그룹 헤더**: 카테고리 라벨 헤더 아래 해당 매장 리스트. 모든 카테고리 동시 노출, 스크롤로 탐색.
- **(b) 필터 칩**: 좌측 상단에 카테고리 칩 노출, 선택 시 해당 카테고리만 표시. `HelpDeskOrderTab` 패턴과 동일.

#### 재사용 가능 자산

| 위치 | 용도 |
|---|---|
| `src/lib/foodCategories.ts:fetchFoodCategories()` | 카테고리 마스터 (`food_categories` 테이블) 조회 — slug/label/sort_order |
| `src/lib/foodCategories.ts:getCategoryColorIndex(slug, categories)` | 5색 사이클 인덱스 반환. CSS 모듈에 `catColor0..catColor4` 클래스 필요 |
| `src/types/festival_extras.ts:FoodBooth.category` | 부스의 카테고리 slug (nullable) |
| `src/pages/admin/helpdesk/HelpDeskOrderTab.tsx:347-380` 부근 | 필터 칩 + 카테고리 컬러 패턴 참조 |
| `src/pages/admin/helpdesk/AdminHelpDesk.module.css` | `.filterChip`, `.filterChipOn`, `.catColor0..4` 클래스 참고 가능 |

#### 구현 메모

- `MenuStep` 의 `useEffect` 안에서 `fetchFoodBooths` 와 `fetchFoodCategories()` 를 `Promise.all` 로 같이 가져오기
- 카테고리가 없는(`category=null`) 부스는 "기타" 그룹 또는 끝에 별도 표시
- 키오스크 디자인 톤 유지: Lucide strokeWidth 1.2, 큰 폰트, 큰 터치 영역
- 선택된 부스 활성 강조는 기존 검정 배경 유지하되 카테고리 컬러를 보조 인디케이터로 (작은 점·라벨 색)

#### 자체 점검

- 카테고리 마스터 비어있을 때 fallback (평면 리스트로 우아하게 회귀)
- 카테고리에 매장이 0개일 때 헤더 자체 숨김
- 옵션 (a) 선택 시 좌측 컬럼이 너무 길어지지 않게 max-height/scroll 처리 (기존 `.boothList` 가 이미 `overflow-y:auto`)
