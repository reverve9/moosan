### A. 손님 결제 401 Unauthorized 핫픽스 (Supabase 직접 적용)

**증상**: 손님 결제 시 `POST /rest/v1/orders 401`, "주문생성실패" 토스트. payments INSERT 는 성공, orders INSERT 만 실패.

**원인 추적**:
- `28_order_number_v2.sql:34` 가 `booth_order_counters` 를 `DROP TABLE … CASCADE` 후 재생성하면서 12 번 마이그레이션 (`12_payments_booth_orders.sql:229,247-248`) 의 `ENABLE ROW LEVEL SECURITY` + permissive 정책이 같이 날아감.
- orders INSERT BEFORE 트리거 `trg_orders_set_number` → `generate_booth_order_number()` (SECURITY INVOKER) → `booth_order_counters` UPSERT 시 anon 권한 없음 → PostgREST 가 401 반환.
- payments 가 통과한 건 그쪽엔 트리거 안에서 다른 테이블 건드리는 게 없어서.

**핫픽스 SQL** (운영 DB 적용 완료, 손님 결제 정상 복구 확인):
```sql
ALTER TABLE booth_order_counters ENABLE ROW LEVEL SECURITY;
CREATE POLICY "booth_order_counters_all" ON booth_order_counters
  FOR ALL USING (true) WITH CHECK (true);
GRANT SELECT, INSERT, UPDATE ON booth_order_counters TO anon, authenticated;
```

**파일 백업**: `_DEV/Seeds/31_booth_order_counters_rls.sql` (gitignored). 멱등하게 작성.

### B. 토스 DUPLICATED_ORDER_ID 영구해결 (Supabase 직접 적용)

**증상**: 토스 결제창 호출 시 `{code: 'DUPLICATED_ORDER_ID', orderId: 'P-00000018'}` — "이미 승인 및 취소가 진행된 중복된 주문번호".

**원인**: `payments.toss_order_id` 가 시퀀스 `payment_toss_order_seq` 로 `P-00000001` 부터 발급. 과거에 `_test_reset_and_day_code_patch.sql:85` 의 `RESTART WITH 1` 가 dev DB 에 돌면서 같은 번호가 두 번째로 토스에 송신됨. 토스는 같은 merchant 의 모든 과거 orderId 를 영구 추적해서 거부.

**해결**: 시퀀스 RESTART 사고에 면역되도록 toss_order_id 포맷에 KST 타임스탬프 끼워넣음.
- 구: `P-00000018` (8자리)
- 신: `P-260507143022-000019` (`P-YYMMDDHH24MISS-NNNNNN`, 21자, 토스 64자 제한 내)

**적용 SQL** (운영 DB 적용 완료):
```sql
CREATE OR REPLACE FUNCTION generate_toss_order_id()
RETURNS TEXT AS $$
BEGIN
  RETURN 'P-'
      || to_char(now() AT TIME ZONE 'Asia/Seoul', 'YYMMDDHH24MISS')
      || '-'
      || LPAD(nextval('payment_toss_order_seq')::TEXT, 6, '0');
END;
$$ LANGUAGE plpgsql;
```

**호환성**: payments.toss_order_id 는 `TEXT UNIQUE`. 기존 `P-00000017` 류 row 와 신 포맷 공존 가능. `findPaymentByTossOrderId` 도 정확 일치 조회라 무관. admin AdminOrders / StatsRevenueTab / CheckoutSuccessPage 모두 문자열 그대로 표시·조회.

**파일 백업**: `_DEV/Seeds/32_toss_order_id_timestamp.sql` (gitignored).
