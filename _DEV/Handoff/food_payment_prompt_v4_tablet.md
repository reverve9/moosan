# 음식문화페스티벌 — 매장 갤럭시탭 운영 반영

> 기존 프롬프트 v1~v3 에 추가되는 내용

---

## 변경 배경

25개 매장 전체에 갤럭시탭 지급 결정. 매장 직원 개인 폰 대신 갤럭시탭으로 부스 대시보드 운영.

---

## 1. Wake Lock 조건부 처리 제거

기존 v2 프롬프트의 Wake Lock은 "Android 지원, iOS 미지원이므로 조건부 처리" 였으나,
갤럭시탭(Android) 고정 환경이므로 조건부 분기 불필요.

```typescript
// 기존 — 조건부
if ('wakeLock' in navigator) {
  await navigator.wakeLock.request('screen');
}

// 변경 — 그냥 실행 (Android 보장)
await navigator.wakeLock.request('screen');
```

## 2. 매장 대시보드 레이아웃 — 태블릿 최적화

### 1단계 (현재): 모바일 기준으로 구현
- `max-width: 600px` 기준 그대로 개발
- 기능 완성 우선

### 2단계 (추후): 태블릿 최적화 추가
- 미디어 쿼리 `min-width: 768px` 분기
- 주문 카드 2열 그리드
- 좌측: 대기 주문 목록 / 우측: 완료 주문 + 당일 매출
- 품절 관리 탭 상단 고정

**⚠️ 지금 당장 태블릿 레이아웃 구현하지 말 것 — Phase 완료 후 별도 작업**

## 3. 사전 세팅 지침 (개발 외 운영 사항)

행사 전날 25대 탭 일괄 세팅:

```
1. 크롬 브라우저 실행
2. 매장 URL 북마크 등록 (/booth/login)
3. 매장별 ID/PW 로그인 완료 상태로 배포
4. 화면 밝기 최대, 자동 잠금 해제 설정
5. Wake Lock 동작 확인
6. 충전 케이블 각 부스 배치
```

---

## 4. 변경 불필요한 항목

아래는 기존 프롬프트 그대로 유지:
- 소리 + 진동 알림 (Android 동일)
- 수동 새로고침 버튼 (네트워크 불안정 대비 유지)
- 연결 끊김 배너 + 자동 재연결 (유지)
- 부스 로그인 / 세션 관리 (변경 없음)
