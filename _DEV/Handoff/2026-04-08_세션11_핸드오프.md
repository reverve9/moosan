## 1. 핵심 결정사항 (이 세션)

| 항목 | 결정 |
|---|---|
| **주문 모델** | 1 Toss 결제 = **N orders (부스 수만큼)**. 푸드코트형. payments 테이블 신규, orders 부스-scope, order_items 는 순수 라인 아이템 |

### Block C — Realtime 필터 이슈 해결

**증상** — 부스 대시보드에서 새 주문 카드가 새로고침 해야 보임. 어드민 모니터는 OK (필터 없음), 손님 `/order/:id` 도 확인 시 반영 OK.

**원인** — `subscribeBoothOrders` 의 `booth_id=eq.X` 는 non-PK 필터. 기본 `REPLICA IDENTITY DEFAULT` 는 PK 만 포함 → UPDATE 이벤트 broadcast 시 old row 에 `booth_id` 없음 → Supabase Realtime 이 filter 매칭 못 해 이벤트 스킵. INSERT 는 new row 에 booth_id 있으니 OK 지만, 결제 흐름은 INSERT(pending) → UPDATE(pending→paid) 라 paid 전이가 UPDATE 로 왔다가 막힘.

**해결** — `_DEV/Seeds/12_payments_booth_orders.sql` 에 `ALTER TABLE ... REPLICA IDENTITY FULL` 3 줄 추가 (payments/orders/order_items). 유저가 즉시 supabase editor 에서 실행 → 해결 확인.

이후 **REPLICA IDENTITY FULL 을 모든 realtime filter 대상 테이블의 기본 규약** 으로 잡음. 13_booth_status.sql 에서도 food_booths 에 동일 설정.
