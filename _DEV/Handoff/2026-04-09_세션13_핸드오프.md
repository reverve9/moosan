### A. 클라이언트 폰트 시스템 cq 통일

**배경**: 클라이언트 앱이 vw clamp(`--text-xs ~ --text-4xl`) + cq(`--text-cq-*`, Festival 페이지만) + px 직접값 3 갈래로 갈려 있었음. 모바일 PWA 에서 viewport ≈ container 라 두 시스템 차이가 거의 없는데 분리되어 있어 통일.

**Phase 1 — 토큰 재정의** (`src/styles/theme/tokens.css`)
- `--text-xs ~ --text-4xl` 정의를 vw clamp → cqw 로 교체. **이름 유지** → 사용처 코드 변경 0
- anchor: **13px @ 380 viewport** (`--text-base = 3.421cqw`)
- 다른 vw clamp 토큰도 같은 방식으로 cqw 교체: `--btn-height-*`, `--btn-padding-x`, `--btn-font-size`, `--icon-sm/md/lg`, `--header-height`, `--bottom-nav-height`
- Festival `--text-cq-*` 는 **그대로 보존** (anchor 13px @ 430)
- 시각 변화 거의 없음 (380~430 viewport 에서 1px 미만 차이)

**Phase 2 — form 토큰 시리즈 신규 + 입력형 페이지 적용**

  --text-form-input:   4.211cqw;  /* 16 @380 ← iOS 줌 회피선 */
  --text-form-label:   3.421cqw;  /* 13 @380 — sectionTitle, label */
  --text-form-helper:  3.158cqw;  /* 12 @380 — 본문, helper */
  --text-form-caption: 2.895cqw;  /* 11 @380 — caption, hint */

- 입력 컴포넌트 (Input/Select/Checkbox/RadioGroup) 는 **어디서 쓰이든 항상 form 토큰**

### D. 부스 PWA 가로 강제 (manifest 분리 + inline script 분기)

**문제**: 단일 `manifest.webmanifest` 가 `orientation: portrait` 라 모든 PWA(손님/부스/어드민) 가 세로 고정. 부스는 태블릿 거치 운영이라 **가로 고정** 필요 (세로 운영은 의미 없음).

**시도했다가 폐기 — Vercel rewrite**
- `vercel.json` 의 `has[host]` 매칭으로 부스 호스트만 manifest URL 을 다른 파일로 분기 시도 (named capture 정규식 + 정확 매칭 fallback)
- 부스 vercel project 의 production 도메인(`moosan-booth.vercel.app`)에서 매칭 동작 안 함 — 핸드오프 12 의 "Vercel 3 → 1 프로젝트 통합 대기" 와 관련된 multi-project 환경 이슈로 추정
- 폐기 후 `vercel.json` 은 원래의 SPA fallback 만 남김

**최종 해결 — index.html inline script 로 client-side 분기**
- `public/booth-manifest.webmanifest` 신규 (`orientation: "landscape"`, name "설악무산축전 부스")
- `index.html` 의 정적 `<link rel="manifest">` **제거** → `<head>` 에 inline script 추가:
  ```html
  <script>
    (function () {
      var host = window.location.hostname.toLowerCase()
      var path =
        host.indexOf('booth.') === 0 || host.indexOf('-booth.') !== -1
          ? '/booth-manifest.webmanifest'
          : '/manifest.webmanifest'
      var link = document.createElement('link')
      link.rel = 'manifest'
      link.href = path
      document.head.appendChild(link)
    })()
  </script>
  ```
- 클라이언트 페이지 영향: 정적 link → script 동적 link 한 단계 차이뿐, **실제 fetch 되는 manifest 와 PWA 동작은 동일**
- `BoothDashboardPage.tsx:87-96` 의 `screen.orientation.lock('landscape')` 가 manifest 와 일관되게 동작 (manifest 가 portrait 였을 때 lock 이 거부되어 무력화됐음)
- 갤럭시탭 안드로이드에서 가로 강제 정상 동작 확인 (사용자 검증)

**참고**: `/manifest.webmanifest` URL 을 브라우저 주소창에 직접 입력해서 열면 **정적 파일 그대로** (portrait) 반환됨. 이건 정상 — PWA 가 실제로 fetch 하는 manifest 는 페이지 로드 후 inline script 가 생성한 link 의 href 임.
