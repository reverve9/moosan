## 2. 본 세션 완료 작업

### 2-1. 픽업 디스플레이 — RAF 기반 끊김 없는 무한 루프 + 속도 슬라이더 ⭐

**위치**: `src/pages/display/DisplayPickup.tsx` + `.module.css`

**문제**: 기존 CSS keyframe (`@keyframes scrollLeft` + `--scroll-distance` 변수) 방식이 카드 추가/삭제·폰트 변경 등으로 `--scroll-distance` 가 mid-cycle 에 갱신되면 진행 중 키프레임이 새 목표값으로 점프 → 끝번호 직후 한참 공백 나오다 이어지는 현상 발생.

**해결**:
- CSS `animation: scrollLeft` 제거, `@keyframes scrollLeft` 삭제
- `useEffect` + `requestAnimationFrame` 으로 트랜스폼 직접 제어 (`trackRef`, `offsetRef`)
- 매 프레임 `offset += speed * dt`, `offset >= wrap (= setWidth + cardGap)` 이면 즉시 `wrap` 빼서 한 프레임 안에 set1→set2 시각 점프 없이 wrap
- 의존성에 `settings.scrollSpeed` 포함 — 속도 변경 시 `offsetRef` 유지하고 RAF 재시작해서 현재 위치 그대로 새 속도로 이어짐
- `will-change: transform` 추가 (GPU 합성)

**컨트롤 패널에 슬라이더 추가**: `스크롤 속도 · {N}px/s` (20~300, step 10, 기본 80). `DisplaySettings.scrollSpeed` 로 localStorage 저장.

**관련 변경**: 기존 상수 `SCROLL_SPEED_PX_PER_SEC` 제거, 인라인 `--scroll-distance/--scroll-duration` CSS 변수 제거.

### 2-3. 키오스크 BoothModal — 결제 동선 어색함 해소 ⭐

**위치**: `src/pages/kiosk/steps/MenuStep.tsx` + `.module.css`

**문제**: BoothModal 안에서 메뉴 담은 직후 결제 가려면 → X 로 모달 닫고 → 위쪽 결제 버튼 클릭. 2-스텝 동선이 한 번 끊겼음.

**해결**:
- **BoothModal 하단에 sticky in-modal 카트 바 추가** — 담긴 게 있을 때만 (`items.length > 0`) 노출. 좌측 카트 요약(개수·금액) 클릭 → 모달 닫고 CartModal 오픈, 우측 "결제 요청" 클릭 → 모달 닫고 Phone 스텝 진행. 부모의 `onGoToPhone`, `setCartOpen` 을 `onPay`/`onOpenCart` props 로 전달.
- **부모 sticky 카트 바를 조건부 렌더** — `items.length > 0` 일 때만. 빈 상태 화면 깔끔하게. 모달 닫고 부스 리스트 뷰에서 결제 진입 경로는 그대로 유지(데드엔드 방지).
- **modalBody 에 `modalBodyFill` 클래스 추가** (`flex: 1; min-height: 0`) — 긴 메뉴 리스트에서 body 가 스크롤되고 카트 바가 모달 바닥에 항상 붙도록. 사용자앱 `FoodSections.module.css` 의 `.modalBody` 는 건드리지 않음 (kiosk 전용 override).

### 2-4. 카트 바 비우기 버튼 (부모 + 모달) ⭐

**위치**: `src/pages/kiosk/steps/MenuStep.tsx` + `.module.css`, `src/pages/kiosk/ResetConfirmModal.tsx`

**문제**: BoothModal 열린 상태에선 헤더 "처음으로" 버튼이 백드롭에 가려져 카트만 비우는 게 2~3 step (모달 닫고 → 헤더, 또는 카트 아이콘 → CartModal → 전체 비우기) 으로 어색했음.

**해결**:
- 부모 sticky 카트 바와 모달 내부 카트 바 좌측에 **trash 아이콘 1탭** 비우기 버튼 추가
- 클릭 시 **`ResetConfirmModal` 띄움** — title/body/confirmLabel override 로 "장바구니를 비울까요?" / "담은 메뉴를 모두 비웁니다. 진행 단계는 그대로 유지돼요." / "비우기"
- Confirm → `clear()` (cartStore) — step 은 건드리지 않음
- 기존 헤더 "처음으로" 시나리오는 ResetConfirmModal props default 유지로 무영향. **단계 리셋(헤더)** vs **카트만 비우기(카트 바)** 의도적으로 구분 (문구로 명시).
