## 1. 세션 18 에서 끝난 것

### A. 전화번호 포맷 전면 통일

**배경**: 기존에 `orders.phone`/`payments.phone` 은 `010-1234-5678` (하이픈 포함), `applications.phone` 은 `01012345678` (하이픈 없음) 으로 혼재. 쿠폰 자동 적용을 전화번호 키로 하려면 통일 필요.

**정책 (최종 확정)**:

| 계층 | 포맷 | 예시 |
|---|---|---|
| DB 저장 | 하이픈 없음 | `01012345678` |
| 입력 UI (타이핑 중) | 하이픈 자동 삽입 | `010-1234-5678` |
| Placeholder | 하이픈 있음 | `010-0000-0000` |
| 디스플레이 (리스트/모달) | 하이픈 있음 | `010-1234-5678` |
| Toss `customerMobilePhone` | 하이픈 없음 | `01012345678` |

- 기존 DB 데이터는 **백필 안 함** (더미라 지울 예정). 혼재해도 `formatPhoneDisplay()` 가 하이픈 유무 관용해서 동일하게 표시.

**`src/lib/phone.ts` 확장**
- `formatPhone(raw)` — 입력 UI 용, 타이핑 중 하이픈 자동 삽입 (기존 동작 유지)
- `normalizePhone(raw)` — 신규, DB/API 저장용. `replace(/\D/g, '').slice(0, 11)`
- `formatPhoneDisplay(raw)` — 신규, 표시용. 숫자만 추출 후 11자리 → `010-1234-5678`, 11자리 아니면 원본 반환
- `isValidPhone(phone)` — 입력 UI 검증용, 하이픈 포함 정규식 유지 (`/^010-\d{4}-\d{4}$/`)
- `saveLastPhone/loadLastPhone` — localStorage (UI prefill 용이라 포맷된 값 그대로)

**제출 경로 normalize 삽입 (DB 저장 직전)**
- `CheckoutPage.tsx` — `createPendingPayment({ phone: normalizedPhone, ... })` + Toss `customerMobilePhone: normalizedPhone` (기존 `.replace(/-/g, '')` 제거)
- `CartPage.tsx` — `fetchPaymentsByPhoneToday(normalizePhone(submittedPhone))` (조회 경로)
- `SurveyForm.tsx` — `submitSurvey({ phone: normalizePhone(form.phone), ... })`
- `ApplyForm.tsx`, `ArtApplyForm.tsx`, `WritingApplyForm.tsx`, `ChoirApplyForm.tsx` — insert 시 phone/parentPhone/teacherPhone 전부 normalize
- `lib/coupons.ts createCouponManually` — defensive normalize on `issued_phone`
- `lib/adminPayments.ts fetchPaymentsList` — `filters.phone` 을 `normalizePhone` 해서 ilike 부분매칭

**디스플레이 경로 formatPhoneDisplay 교체**

```ts
async function submitSurvey(input) {

    throw new DuplicateSurveyCouponError()
  }
  // 2) surveys insert (기존 phone+festival unique 제약 유지)
  // 3) issueSurveyCoupon(input.phone)
  return { survey, coupon }
}
```
