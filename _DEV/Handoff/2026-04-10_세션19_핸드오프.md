#### A-1. 공통 인프라 (신규 파일)

**`src/hooks/useWakeLock.ts`**
- Screen Wake Lock API 래퍼. `useWakeLock(enabled=true)` 로 호출.
- 마운트 시 `navigator.wakeLock.request('screen')` 요청
- `visibilitychange` 리스너 등록 — 탭 복귀 시 자동 재요청 (브라우저가 탭 비활성 시 자동 release)
- 언마운트 시 수동 release, cancel 플래그로 race 처리
- 미지원 브라우저 (iOS Safari 등) / HTTP / 실패 전부 **조용히 no-op**
- 탭 자체 keep-awake 설정이 1차 방어선, 이 훅은 보조

**`src/lib/audioCue.ts`**
- `playSound(type: 'alarm' | 'alert', repeat=3)` — `/sounds/order_alarm.mp3` 또는 `/sounds/order_alert.mp3` 를 `new Audio()` 로 생성해 `await` 기반 순차 재생
- `ended`/`error`/`play().catch` 전부 silently resolve → 호출부 try/catch 불필요
- `unlockAudio()` — iOS Safari autoplay 정책 우회용. 사용자 제스처 핸들러 안에서 한 번 호출해 AudioContext unlock. 무음(`volume=0`) 짧은 재생 후 pause.
- 사운드 파일은 사용자가 직접 `public/sounds/` 에 배치 완료.

**`src/components/admin/AdminAlertContext.tsx`** — 어드민 전역 알림 상태
- 상수: `WARN_SECONDS = 60` (1분 초과 주황) / `ALERT_SECONDS = 120` (2분 초과 빨강)
- `elapsedSeconds(iso, nowMs)` 유틸 export
- `AdminAlertProvider` — AdminLayout 인증 이후에만 마운트
  - 초기 로드: `fetchMonitorSummary` 1회
  - Realtime: `subscribeMonitor('admin-alert-context')` 로 orders 변경 감지 → 재조회
  - 1초 `setInterval` tick → `now` state 업데이트
  - useMemo 로 `totalPending`, `warnCount` (1~2분), `alertCount` (2분+) 집계
  - `refetch()` 수동 새로고침 — AdminMonitor 의 새로고침 버튼용 (provider 가 refreshing state 소유)
- `useAdminAlert()` 훅 — context 없으면 throw
- **기존 AdminLayout/AdminMonitor 가 각자 `fetchMonitorSummary/subscribeMonitor/1초 tick` 돌리던 이중 구독 제거** → 단일 소스

#### A-2. 부스 (소리·진동·Wake Lock)

**`src/pages/booth/BoothLoginPage.tsx`**
- `handleSubmit` 첫 줄에 `unlockAudio()` — 로그인 버튼 클릭 제스처에서 AudioContext unlock. 이후 대시보드에서 자동 재생이 iOS Safari 에서도 통과.

**`src/pages/booth/BoothDashboardPage.tsx`**
- `useWakeLock(true)` 적용 (DashboardInner)
- `ALERT2_SECONDS = 120` 상수 추가 (기존 `ALERT_SECONDS = 60` 유지)
- `vibrateSafe(pattern: number[])` 헬퍼 — `navigator.vibrate` 없거나 throw 나면 조용히 무시
- **새 주문 수신**: 기존 `subscribeBoothOrders.onOrderPaid` 콜백에 `playSound('alarm', 3)` + `vibrateSafe([300,100,300,100,300])` 추가. 초기 snapshot 은 `refetch()` 경로라 이 콜백 안 들어옴 (pending→paid UPDATE 이벤트만 잡힘) → **초기 로드 알람 중복 걱정 없음**
- **1분/2분 초과 전이 감지**: 기존 1초 tick `now` state 에 의존하는 별도 useEffect 추가
  - `alertedIdsRef = { one: Set<string>, two: Set<string> }` 으로 주문별 1회 알람 강제
  - `status === 'paid' && !ready_at && !cancelled_at` 인 주문만 검사
  - 각 임계를 처음 넘는 순간 `playSound('alert', 3)` + `vibrateSafe([500,200,500])` (2분 초과는 `[500,200,500,200,500]`)
  - 완료/취소된 주문은 Set 에서 삭제 (메모리 누수 방지)

#### A-4. 실시간 반영 경로 (사용자 "특히 신경써" 지시)

모든 알림 소비자가 **AdminAlertContext 단일 소스** 를 구독하므로 다음 이벤트 발생 시 **동시 업데이트**:

1. 주문 상태 변경 → `subscribeMonitor` realtime → `fetchMonitorSummary` 재실행 → `summaries` state 업데이트 → useMemo 재계산 → 전체 소비자 리렌더
2. 시간 경과 → 1초 tick → `now` state 업데이트 → useMemo 재계산 → 임계 넘는 주문 즉시 배지/배너/카드/document.title 전부 반영

### B. 공지사항 에디터 리팩토링 — 마크다운 폐기
