### Block F — subdomain 분리 (B안) ★ 핵심 인프라

운영 시 가장 큰 리스크였던 "URL 실수 / 도메인 경계 불명확" 해결. **one app, hostname 분기** 패턴.

- **`src/App.tsx` 전면 재구성**
  - `getAppMode()` — `window.location.hostname` 보고 `'booth' | 'admin' | 'customer'` 반환
    - `host.startsWith('booth.')` → booth
    - `host.startsWith('admin.')` → admin
    - 그 외 → customer
  - `APP_MODE` 모듈 상수 (한 번만 평가)
  - 모드별 라우트 트리 분리: `BoothRoutes()` / `AdminRoutes()` / `CustomerRoutes()` 컴포넌트
  - **booth 모드**: `/`, `/login`, `/dashboard`, `*` → `/login` (catch-all)
  - **admin 모드**: `/` (AdminLayout 루트) + `applications/festivals/programs/food/booth-accounts/monitor/notices` 자식, `*` → `/`
  - **customer 모드**: 기존 / + Layout 자식들
  - `FloatingInstallButton` 도 customer 모드에서만 mount
- **`src/pages/booth/BoothLoginPage.tsx`**: `navigate('/booth/dashboard')` → `navigate('/dashboard')`
- **`src/pages/booth/BoothDashboardPage.tsx`**: `navigate('/booth/login')` → `navigate('/login')`
- **`src/components/admin/AdminLayout.tsx`**:
  - `NAV_GROUPS` path 들 `/admin/X` → `/X`
  - `MONITOR_PATH = '/admin/monitor'` → `'/monitor'`
  - logo onClick `navigate('/admin')` → `navigate('/')`
  - "사이트로 이동" 버튼: `window.open('/', '_blank')` → 호스트네임에서 `admin.` prefix 떼고 손님 도메인 새 탭으로 (`window.location.hostname.replace(/^admin\./, '')`)
- **`vite.config.ts`**: `server: { host: true }` — booth.localhost / admin.localhost 등 서브도메인 접근 허용

**dev 접근 방법** (macOS 는 `*.localhost` 자동 해석):
```
http://localhost:5173/              ← 손님
http://booth.localhost:5173/        ← 가맹점 (자동으로 /login 으로)
http://admin.localhost:5173/        ← 어드민
```
혹은 vercel dev 사용 시 포트 3000.

윈도우/리눅스는 `/etc/hosts` 에:
```
127.0.0.1  booth.localhost admin.localhost
```

### ❌ 미검증 / 의심 (다음 세션 1번 우선 디버그)

#### **이슈 1: 어드민 실시간 모니터에 새 주문이 안 뜸** ⚠️
- 부스 대시보드에는 **잘 보임**
- 어드민 `/monitor` 페이지에는 **새 주문 카드가 안 나타남**
- AdminLayout 사이드바 빨간 배지도 같이 안 올라옴
- **가장 유력한 원인**: 머신 교체 후 Supabase Realtime publication 확인 미완료. `supabase_realtime` publication 에 `orders`, `order_items` 가 등록돼 있는지 Studio → Database → Replication 에서 확인 필요. 빠져있으면:
  ```sql
  ALTER PUBLICATION supabase_realtime ADD TABLE orders, order_items;
  ```
- **2차 디버그**: 어드민 모니터 우상단 [새로고침] 버튼을 수동으로 눌렀을 때 — 카운트가 올라오면 realtime 만 stale, 안 올라오면 쿼리/RLS 문제 (근데 booth 가 같은 쿼리 패턴으로 보고 있어서 RLS 문제는 거의 불가능)
- **3차 디버그**: 브라우저 콘솔에서 supabase realtime 채널 구독 상태 확인. `admin-monitor-page-items` / `admin-monitor-page-orders` / `admin-layout-monitor-items` / `admin-layout-monitor-orders` 4개 채널이 SUBSCRIBED 상태여야 함

#### **이슈 2: 부스 확인 버튼 → 어드민 카운트 즉시 반영 안 됨**
- 위 이슈 1과 같은 원인. realtime UPDATE on order_items 가 어드민에 안 닿음. publication 만 잡으면 자동 해결될 가능성 큼

#### **이슈 3: 부스 대시보드 alert 풀림 동작 검증**
- 이번 세션 마지막에 fix 한 `card.status === 'waiting' && ...` 조건이 실데이터에서 잘 동작하는지 확인 필요. **확인 클릭 → 카드 빨강 (cardAlert) 풀리고 → border 그린 (inProgress) 로 바뀌어야 함**
