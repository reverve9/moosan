# Handoff — 2026-04-06 세션 5

## 작업 배경
- 모바일에서 폰트가 작게 느껴진다는 피드백
- 분석 결과: **YouthPage / ProgramAccordion / PageTitle** 이 `cqw` 단위로 폰트 사이즈를 직접 하드코딩 → `max-width 600px container` 기준이라 모바일(360~390px)에서 비례 축소되어 작아지고 있었음
- `tokens.css` 의 `--text-*` 를 올려도 위 파일들에는 전혀 반영 안 됨 (토큰 사용 X)

## 방향 결정
- **cqw 비례 효과 유지가 최우선** — container 완전 비례여야 모든 너비에서 동일한 줄바꿈·레이아웃 유지
- **clamp 추가 금지** — min 경계에 걸리면 폰트만 고정되고 padding/gap 은 계속 cqw 로 줄어들어 줄바꿈 스위칭·카드 높이 점프 발생
- 해결책: **cqw 값 자체를 토큰 변수에 묶기** → 효과 100% 보존, 구조만 중앙화
- 이후 화면 보면서 토큰 정의 한 줄만 손보면 YouthPage 전역 일괄 조정

## 참고
- `--text-cq-*` 네임스페이스는 **600px container 안에서만** 사용 — 컨테이너 바깥에서는 cqw 기준이 맞지 않음
- 기존 `--text-*` (vw 기반 clamp) 와는 별도 체계. 홈페이지 섹션·Header·BottomNav·Button 은 vw clamp 그대로
- 폰트를 Text 컴포넌트로 교체하지 않은 이유: Text 는 `--text-*` 기반이라 cqw 효과 없음. YouthPage 는 CSS 모듈에서 `--text-cq-*` 직접 사용이 맞음
