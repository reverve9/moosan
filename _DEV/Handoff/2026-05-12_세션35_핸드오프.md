## 6. 알려진 제약 / 운영 메모

### 6-1. legacy 토스 환불 분기 유지
`api/orders/cancel.ts` + `api/payments/cancel.ts` 에 legacy 토스 분기 코드 잔류. 행사 종료 후 별도 세션:
- `TOSS_SECRET_KEY` env 제거
- 토스 분기 코드 블록 삭제
- DB legacy 행 정리 (`pg_provider IS NULL AND payment_key IS NOT NULL`)

### 6-2. 부분 취소 Noti 수동 처리
`api/cookiepay/noti.ts` 가 부분 취소를 받으면 `payments.refunded_amount` 만 누적 + `console.error` 알람. orders 는 운영진 판단. 자주 발생하면 Sentry 등 알람 시스템 연동 검토.

### 6-3. TOKEN 캐시 격리
`api/_lib/cookiepay.ts` 의 `_tokenCache` 는 Vercel function instance 별 격리. 환불 빈도가 높으면 캐시 적중률 낮음. 단기 무방.

### 6-4. sandbox 키와 라이브 키 분리
- 콘솔 자체에서 환경 토글 + 환경별 키 발급. 키 5개 모두 환경 일치 필요 (mismatch 시 "API_ID 값이 올바르지 않습니다" RESULTCODE 9999).
- `COOKIEPAY_SANDBOX` env 와 `index.html` SDK URL 두 군데 모두 동시 일치 필요.
  - 현재 상태: `index.html` = www (live), env `COOKIEPAY_SANDBOX=false` 권장.
  - 샌드박스 복귀 시: `index.html` 두 줄 sandbox 로 + env `true` + 키 5개 sandbox 값으로 교체.

### 6-5. 샌드박스 한계 (검증 시 미사용 결정 배경)
- 샌드박스에서 실 카드 사용 시 진짜 결제 발생 (가청구). 카드사 정산 사이클 전 환불 가능.
- 카카오페이/네이버페이는 샌드박스 미지원 → 라이브에서만 검증 가능.
- 결론: dev 부터 라이브 도메인 + 실 카드 소액 (1,000~2,000원) + 즉시 환불 방식 채택.

### 6-6. 쿠키페이 측 확인 사항 (2026-05-12)
- 카카오/네이버페이도 동일 환불 API (`/api/cancel`) 처리 가능 → B안(수동 환불 정책) 폐기
- 매뉴얼 본문 "신용카드/계좌이체만 환불" 표현은 표면적 — 실 운영상 간편결제 환불 정상 동작
- 영업담당 답변 기준이라 행사 후 실 정산까지 모니터링 필요
