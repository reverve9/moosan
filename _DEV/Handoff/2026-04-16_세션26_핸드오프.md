### A. 부스 대시보드 `await refetch()` 제거 → 낙관적 업데이트

- 파일: `src/pages/booth/BoothDashboardPage.tsx`
- GNfesta 적용 패턴을 무산에 동일 적용
- 변경 대상 핸들러 3개:

| 핸들러 | 변경 전 | 변경 후 |
|--------|---------|---------|
| `handleConfirmWithTime` | `await refetch()` | `setData()` — status='confirmed', confirmed_at, estimated_minutes 즉시 반영 |
| `handleReady` | `await refetch()` | `setData()` — ready_at 즉시 반영 |
| `handleCancelConfirm` | `await refetch()` | `setData(prev => prev.filter(...))` — 카드 즉시 제거 |

- 원리: DB 변경 후 Supabase Realtime 구독의 `onChange` 콜백이 `void refetch()` 를 자동 트리거하므로 명시적 `await refetch()` 불필요
- 각 `useCallback` 의 deps 배열에서 `refetch` 제거
- 체감: 버튼 클릭 → UI 반영까지 0.5~1초 단축 (Toss API + DB + refetch 대기 제거)
