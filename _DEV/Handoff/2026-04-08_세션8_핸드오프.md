## 2. `vercel dev` 첫 실행 시 vite 폭발 — `vercel.json` rewrite 패턴 수정

**증상**: `vercel dev` 띄우자마자 vite 가 `index.html:9:32` 에서 syntax error. `@react-refresh`, `client` 도 모두 500.

**원인**: `vercel.json` 의 SPA fallback rewrite 패턴이 너무 광범위:
```json
"source": "/((?!api/).*)"
```
프로덕션 빌드에선 정적 파일이 rewrite 보다 먼저 매칭돼서 문제 없는데, **`vercel dev` 환경에선 vite 가 dev 모드로 띄우는 가상 경로** (`/src/main.tsx`, `/@vite/client`, `/@react-refresh` 등) 가 전부 `/index.html` 로 rewrite 되면서 vite 가 JS 모듈인 줄 알고 HTML 을 import-analysis 로 파싱 시도 → 폭발.

**수정**: `vercel.json` rewrite 패턴에 vite dev 경로 제외 추가
```json
"source": "/((?!api/|src/|node_modules/|@|.*\\.).*)"
```
- `api/` (이전과 동일)
- `src/`, `node_modules/` (vite dev 가 서빙)
- `@` 시작 (`@vite/client`, `@react-refresh`, `@id/...`)
- `.` 포함 (확장자 있는 파일 — `.tsx`, `.css`, `.svg`, `.png`, `manifest.webmanifest`, `sw.js`)

→ 프로덕션 라우팅도 그대로 동작 (SPA 경로 `/cart`, `/program/food`, `/order/123` 는 점/@/src 없으니 정상 rewrite)

## 3. 결제 모달 viewport 잘림 → `windowTarget: 'self'` 추가

**증상**: 모바일 폭에서 토스 결제창이 viewport 보다 길어서 하단 결제 버튼이 화면 밖으로 잘림. 무이자할부 안내 누르면 모달 위에 모달 떠서 더 어색.

**삽질 1 (실패)**: `.layout` 의 `container-type: inline-size` 가 `position: fixed` 자손의 containing block 을 깨는 거라고 가설 → `.layout` → `.main` 으로 옮김 → 동일 (실제론 토스 모달이 body 직속이라 무관). **나중에 원위치 복원함**.

**삽질 2 (실패)**: chrome `--page-padding-x` 토큰을 12px → 20px 로 변경. 사용자 의도 오해 (사용자는 결제 모달 패딩 얘기였고 chrome 패딩 얘기가 아니었음). **나중에 원위치 복원함**.

**진짜 원인**: 레거시 SDK `@tosspayments/payment-sdk` 의 `requestPayment` 가 `windowTarget` 옵션을 받음:
- `'iframe'` (기본) — 같은 페이지에 iframe 오버레이 (모바일 viewport 제약 받음)
- `'self'` — `window.location` 으로 toss.im 풀페이지 이동

`CheckoutPage.tsx` 의 `requestPayment` 호출에 `windowTarget: 'self'` 추가.

**중요한 데스크탑 폴백**: 토스 SDK 가 user-agent 보고 모드 결정. **데스크탑 (Chrome DevTools 모바일 emulation 포함) 에선 'self' 무시하고 iframe 폴백**. 그래서 데스크탑 디버깅 시엔 viewport 잘림이 여전히 보임. 하지만 **실 모바일 디바이스에선 'self' 정상 발동** → toss.im 풀스크린 → success url 복귀. **실 모바일 검증 완료, 무문제**.

→ 메모리 `project_toss_payment_window_target.md` 신규 생성, MEMORY.md 인덱스 등록

```

│   ├── 부스별 그룹 + 수량 stepper + 삭제
```
